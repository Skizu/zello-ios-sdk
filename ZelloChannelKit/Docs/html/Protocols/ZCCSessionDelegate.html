– sessionDidConnect:

Called when the session finishes connecting to the server and channel

- (void)sessionDidConnect:(ZCCSession *)session

Parameters

session

the session that has just connected

Discussion

After sessionDidConnect: is called, the connection to the server is fully established. You can now call -startVoiceMessage to start speaking to the channel, and will receive incoming voice messages when other users speak to the channel.

sessionDidConnect: is called after an automatic reconnect, so be aware that your delegate may see sessionDidConnect: called multiple times without sessionDidDisconnect: being called. See session:willReconnectForReason: for more about automatic reconnection.

– sessionDidDisconnect:

Called when the session finishes disconnecting from the server

- (void)sessionDidDisconnect:(ZCCSession *)session

Parameters

session

the session that has disconnected

Discussion

If the session is disconnected due to a network change or other unexpected event, this method is not called, and session:willReconnectForReason: is called instead. In that case, the session automatically attempts to reconnect. You can prevent automatic reconnect attempts by implementing session:willReconnectForReason: and returning NO.

– session:willReconnectForReason:

Called when the session has become unexpectedly disconnected.

- (BOOL)session:(ZCCSession *)session willReconnectForReason:(ZCCReconnectReason)reason

Parameters

session

the session that has disconnected
reason

The reason the session was disconnected

Return Value

YES to allow the reconnect attempt to continue, NO to prevent the session from attempting to reconnect.

Discussion

When the session becomes unexpectedly disconnected from a network change or other event, it will automatically attempt to reconnect with a randomized backoff delay. You can prevent the session from reconnecting by implementing this method and returning NO.

– session:incomingVoiceWillStart:

Implement this method to perform custom handling of incoming voice data

- (nullable ZCCIncomingVoiceConfiguration *)session:(ZCCSession *)session incomingVoiceWillStart:(ZCCIncomingVoiceStreamInfo *)streamInfo

Parameters

session

the session that is about to open a new incoming voice stream
streamInfo

an object describing the voice source. You can use this information to determine whether to override the default audio handling.

Return Value

a configuration object specifying a custom voice receiver to handle incoming voice data. If you return nil instead of a configuration object, the stream will play through the current audio output route as normal.

Discussion

If this method is implemented by your session delegate, you can override the Zello Channels SDK’s default processing of incoming voice streams. This method will be called when another user begins speaking on the channel. The streamInfo object describes the channel and the user who has begun speaking. Your implementation can return nil to tell the Zello Channels SDK to play the incoming stream through the device speaker. If you want to perform different handling of the audio, you can return a ZCCIncomingVoiceConfiguration object instead, with a reference to a custom ZCCVoiceReceiver.

If this method is not implemented or returns nil, the Zello Channels SDK will perform its default incoming voice handling and play the audio through the current output audio route.
