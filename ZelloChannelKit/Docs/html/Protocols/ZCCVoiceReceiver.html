ZCCVoiceReceiver Protocol Reference

Conforms to NSObject
Declared in ZCCIncomingVoiceConfiguration.h

Overview

Implement ZCCVoiceReceiver to provide custom handling for incoming voice data.

Your voice receiver will receive a prepareWithAudioDescription:stream: message when the incoming stream has finished opening. Then it will receive receiveAudio:stream: messages repeatedly as new audio comes in from the channels server. When the stream is closing, it will receive a stopReceivingAudio: message.

Warning: All ZCCVoiceReceiver methods are called on an arbitrary dispatch queue.

– prepareWithAudioDescription:stream: required method

Called when an incoming stream opens

- (void)prepareWithAudioDescription:(AudioStreamBasicDescription)description stream:(ZCCIncomingVoiceStream *)stream

Parameters

description

Describes the format of the data in the buffers that you will receive in receiveAudio:stream: calls. Save this object so you know how to process the audio data.
stream

the stream that has just opened

Discussion

When an incoming stream finishes opening, you will receive a prepareWithAudioDescription:stream: message. Use the description to prepare your audio handling code.
